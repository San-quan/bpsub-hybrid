# Directory Structure

This document describes the purpose of the main directories in this repository.

## Core Directories

### `public/`
Static assets and HTML files for the web interface. Contains the public-facing UI components.

### `src/`
Source code for the application logic. Contains library code and modules used by the worker.

### `telegram-bot/`
Telegram bot integration code and configuration. Handles notification system and bot management.

### `tests/`
Test files and test utilities. Contains unit tests and integration tests for the project.

### `workspace/`
Development workspace files, scripts, and configuration samples. Contains helper scripts and development tools.

### `.github/`
GitHub-specific configurations including:
- CI/CD workflows
- Issue and PR templates
- Dependabot configuration
